## Design Context

### Users
Vishal (product designer, Mumbai, actively job-hunting for product companies — April 2026). Private tool. Used during short bursts when updating the resume for a new application, and longer sessions when restructuring sections or rewriting bullets. Exported PDF is the artifact that lands in a recruiter's inbox or ATS — the builder itself is never seen by anyone else.

### Brand Personality
Tool-like, efficient, keyboard-first, quiet. Linear / Raycast / Arc adjacent. Zero marketing energy. No "productivity app" clichés. The builder disappears so the work shows through.

### Aesthetic Direction
- **Theme**: Dark. Standard for this tool category (Linear, Raycast, Figma sidebars) and Vishal edits for long stretches.
- **Type**: Geist Sans + Geist Mono. Mono reserved for keyboard hints, section counts, PDF export metadata — never body.
- **Color**: Warm-neutral dark palette tinted slightly toward cool-blue to read as "tool." Single restrained accent (muted amber) reserved for primary save/export actions — the 10% in 60-30-10.
- **Density**: Medium. Tighter than marketing, looser than a code editor. Rhythm through spacing variation, not borders.
- **Preview**: The resume preview pane is a separate visual world — warm paper (not pure white), navy heading accent, sans-only body. Refined iteration of Vishal's current PDF layout.
- **Anti-references**: Notion (too soft), Typeform (too conversational), generic resume builders (too templated), Canva (too decorative), any "AI productivity" glow-on-dark aesthetic.

### Design Principles
1. **The tool disappears.** Editor chrome is quiet so the resume preview is the focal point.
2. **One kind of surface.** Flat panels with 1px dividers. No nested cards, no drop shadows as decoration.
3. **Keyboard is a first-class input.** Tab order, shortcuts for save/export/section-jump, visible shortcut hints in mono.
4. **The PDF is the product.** Every rendering decision in the preview must match the exported PDF exactly — no surprise on download.
5. **Restraint over decoration.** No gradients, no glows, no icon walls. Hierarchy comes from type weight and spacing.
